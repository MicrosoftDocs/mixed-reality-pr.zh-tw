<span data-ttu-id="84437-140">若要正確轉譯透明或半透明的物件，必須使用存在於背景中的任何物件來排序和混合。</span><span class="sxs-lookup"><span data-stu-id="84437-140">For transparent or translucent objects to render correctly they must be sorted and blended with any objects which exist in the background.</span></span> <span data-ttu-id="84437-141">透明物件的排序具有適度的 CPU 成本，因此混合具有相當大的 GPU 成本，因為它不允許 GPU 透過 z 剔除來執行隱藏的介面移除 (例如</span><span class="sxs-lookup"><span data-stu-id="84437-141">Sorting of transparent objects has a modest CPU cost, blending has considerable GPU cost because it does not allow the GPU to perform hidden surface removal via z-culling (i.e</span></span> <span data-ttu-id="84437-142">深度測試) 。</span><span class="sxs-lookup"><span data-stu-id="84437-142">depth testing).</span></span> <span data-ttu-id="84437-143">不允許隱藏的介面移除會增加需要針對最終轉譯圖元計算的作業數目，因此會對填滿率條件約束造成更大的壓力。</span><span class="sxs-lookup"><span data-stu-id="84437-143">Not allowing hidden surface removal increases the number of operations that need to be computed for the final rendered pixel, and thus puts more pressure on fill rate constraints.</span></span>

## <a name="design-guidelines"></a><span data-ttu-id="84437-148">設計指導方針</span><span class="sxs-lookup"><span data-stu-id="84437-148">Design guidelines</span></span>
<span data-ttu-id="84437-149">**使用不透明的 UI 背景**</span><span class="sxs-lookup"><span data-stu-id="84437-149">**Use opaque UI background**</span></span>

<span data-ttu-id="84437-152">MRTK 中的解決方案-Unity： https://microsoft.github.io/MixedRealityToolkit-Unity/Documentation/hologram-stabilization.html#depth-buffer-sharing-in-unity</span><span class="sxs-lookup"><span data-stu-id="84437-152">Solutions within MRTK-Unity: https://microsoft.github.io/MixedRealityToolkit-Unity/Documentation/hologram-stabilization.html#depth-buffer-sharing-in-unity</span></span>

<span data-ttu-id="84437-154">**將受影響的圖元數目降至最低**</span><span class="sxs-lookup"><span data-stu-id="84437-154">**Minimize the number of pixels affected**</span></span>
