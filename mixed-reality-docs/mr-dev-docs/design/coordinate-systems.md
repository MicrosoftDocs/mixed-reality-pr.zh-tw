## <a name="spatial-coordinate-systems"></a><span data-ttu-id="eeb24-190">空間座標系統</span><span class="sxs-lookup"><span data-stu-id="eeb24-190">Spatial coordinate systems</span></span>

<span data-ttu-id="eeb24-191">所有3D 圖形應用程式都會使用 [笛卡兒座標系統](https://docs.microsoft.com/windows/uwp/graphics-concepts/coordinate-systems) 來表示其所轉譯之虛擬世界中物件的位置和方向。</span><span class="sxs-lookup"><span data-stu-id="eeb24-191">All 3D graphics applications use [Cartesian coordinate systems](https://docs.microsoft.com/windows/uwp/graphics-concepts/coordinate-systems) to reason about the positions and orientations of objects in the virtual worlds they render.</span></span> <span data-ttu-id="eeb24-192">這類座標系統會建立3個垂直軸，以放置物件： X、Y 和 Z 軸。</span><span class="sxs-lookup"><span data-stu-id="eeb24-192">Such coordinate systems establish 3 perpendicular axes along which to position objects: an X, Y, and Z axis.</span></span>

<span data-ttu-id="eeb24-193">在 [混合的現實](../discover/mixed-reality.md)情況下，您的應用程式會導致虛擬和實體座標系統的原因。</span><span class="sxs-lookup"><span data-stu-id="eeb24-193">In [mixed reality](../discover/mixed-reality.md), your apps will reason about both virtual and physical coordinate systems.</span></span> <span data-ttu-id="eeb24-194">Windows 呼叫的座標系統在實體世界中具有真正意義的 **空間座標系統**。</span><span class="sxs-lookup"><span data-stu-id="eeb24-194">Windows calls a coordinate system that has real meaning in the physical world a **spatial coordinate system**.</span></span>
