<span data-ttu-id="9c92f-167">原始 arcade 機器上的拉杆會對應到值的小數位數，移動杆的移動越多，就會有更多天生套用至出貨。</span><span class="sxs-lookup"><span data-stu-id="9c92f-167">The lever on the original arcade machine mapped to a scale of values, the higher the lever was moved the more thrust was applied to the ship.</span></span> <span data-ttu-id="9c92f-168">這裡有一項重要的細微之處，就是使用者可以將控制權交給控制權，並維持所需的價值。</span><span class="sxs-lookup"><span data-stu-id="9c92f-168">An important nuance to point out here is the user can take their hand off of the control and maintain a desired value.</span></span> <span data-ttu-id="9c92f-169">我們可以有效地使用點擊和拖曳行為來達到相同的結果。</span><span class="sxs-lookup"><span data-stu-id="9c92f-169">We can effectively use tap-and-drag behavior to achieve the same result.</span></span> <span data-ttu-id="9c92f-170">天生值從零開始。</span><span class="sxs-lookup"><span data-stu-id="9c92f-170">The thrust value starts at zero.</span></span> <span data-ttu-id="9c92f-171">使用者可點擊和拖曳以增加值。</span><span class="sxs-lookup"><span data-stu-id="9c92f-171">The user taps and drags to increase the value.</span></span> <span data-ttu-id="9c92f-172">他們可能會讓我們繼續維護。</span><span class="sxs-lookup"><span data-stu-id="9c92f-172">At that point they could let go to maintain it.</span></span> <span data-ttu-id="9c92f-173">任何點拖曳手勢值變更都會是原始值的差異。</span><span class="sxs-lookup"><span data-stu-id="9c92f-173">Any tap-and-drag gesture value change would be the delta from the original value.</span></span>

<span data-ttu-id="9c92f-175">這會比較複雜一點。</span><span class="sxs-lookup"><span data-stu-id="9c92f-175">This is a little more tricky.</span></span> <span data-ttu-id="9c92f-176">讓全像「旋轉」按鈕，以提供更豐富的體驗。</span><span class="sxs-lookup"><span data-stu-id="9c92f-176">Having holographic “rotate” buttons to tap makes for a terrible experience.</span></span> <span data-ttu-id="9c92f-177">沒有實際的控制項可供使用，因此其行為必須來自操作代表 lander 的物件，或使用 lander 本身。</span><span class="sxs-lookup"><span data-stu-id="9c92f-177">There isn’t a physical control to leverage, so the behavior must come from manipulation of an object representing the lander, or with the lander itself.</span></span> <span data-ttu-id="9c92f-178">我們會利用點一下和拖曳方法，讓使用者以他們想要的方向有效地「推送和提取」。</span><span class="sxs-lookup"><span data-stu-id="9c92f-178">We came up with a method using tap-and-drag which enables a user to effectively “push and pull” it in the direction they want it to face.</span></span> <span data-ttu-id="9c92f-179">當使用者按下並按住時，開始手勢的空間點就會變成旋轉的原點。</span><span class="sxs-lookup"><span data-stu-id="9c92f-179">Any time a user taps and holds, the point in space where the gesture was initiated becomes the origin for rotation.</span></span> <span data-ttu-id="9c92f-180">從原點拖曳可將手平移的差異轉換 (X、Y、Z) ，並將其套用至 lander 旋轉值的差異。</span><span class="sxs-lookup"><span data-stu-id="9c92f-180">Dragging from the origin converts the delta of the hand's translation (X,Y,Z) and applies it to the delta of the lander's rotation values.</span></span> <span data-ttu-id="9c92f-181">或者更簡單地說，將 *左拖曳 <-> 右邊、向上 <-> 向下、在空間中向前 <->，則會據以旋轉出貨* 。</span><span class="sxs-lookup"><span data-stu-id="9c92f-181">Or more simply, *dragging left <-> right, up <-> down, forward <-> back in spaces rotates the ship accordingly* .</span></span>

<span data-ttu-id="9c92f-182">由於 HoloLens 可以追蹤兩個手，因此可以在天生由左方控制時，將旋轉指派給右手邊。</span><span class="sxs-lookup"><span data-stu-id="9c92f-182">Since the HoloLens can track two hands, rotation can be assigned to the right hand while thrust is controlled by the left.</span></span> <span data-ttu-id="9c92f-183">Finesse 是此遊戲中成功的驅動因素。</span><span class="sxs-lookup"><span data-stu-id="9c92f-183">Finesse is the driving factor for success in this game.</span></span> <span data-ttu-id="9c92f-184">這些互動的 *風格* 是絕對最高的優先順序。</span><span class="sxs-lookup"><span data-stu-id="9c92f-184">The *feel* of these interactions is the absolute highest priority.</span></span> <span data-ttu-id="9c92f-185">尤其是在 tactile 深度的內容中。</span><span class="sxs-lookup"><span data-stu-id="9c92f-185">Especially in context of tactile immersion.</span></span> <span data-ttu-id="9c92f-186">如果出貨的反應太快，可能會不必要地進行操作，但速度太慢時，使用者必須在出貨時「推播並提取」一段滑雪長的時間。</span><span class="sxs-lookup"><span data-stu-id="9c92f-186">A ship that reacts too quickly would be unnecessarily difficult to steer, while one too slow would require the user to “push and pull” on the ship for an awkwardly long amount of time.</span></span>

### <a name="adding-input-for-game-controllers"></a><span data-ttu-id="9c92f-187">新增遊戲控制器的輸入</span><span class="sxs-lookup"><span data-stu-id="9c92f-187">Adding input for game controllers</span></span>

<span data-ttu-id="9c92f-188">雖然 HoloLens 上的手手勢提供了一種精細控制的新穎方法，但仍有某些缺乏「true」 tactile 的意見反應可從類比控制項獲得。</span><span class="sxs-lookup"><span data-stu-id="9c92f-188">While hand gestures on the HoloLens provide a novel method of fine-grain control, there is still a certain lack of 'true' tactile feedback that you get from analog controls.</span></span> <span data-ttu-id="9c92f-189">連接 Xbox 遊戲控制器可讓您回頭 physicality，同時利用控制杆來保有更精細的控制。</span><span class="sxs-lookup"><span data-stu-id="9c92f-189">Connecting an Xbox game controller allows to bring back this sense of physicality while leveraging the control sticks to retain fine-grain control.</span></span>

<span data-ttu-id="9c92f-190">有多種方式可將相對的直接控制項配置套用至 Xbox 控制器。</span><span class="sxs-lookup"><span data-stu-id="9c92f-190">There are multiple ways to apply the relatively straight-forward control scheme to the Xbox controller.</span></span> <span data-ttu-id="9c92f-191">由於我們想要盡可能保持最接近原始 arcade 的設定， **天生** 對應最適合 [觸發程式] 按鈕。</span><span class="sxs-lookup"><span data-stu-id="9c92f-191">Since we're trying to stay as close to the original arcade set up as possible, **Thrust** maps best to the trigger button.</span></span> <span data-ttu-id="9c92f-192">這些按鈕是類比控制項，也就是它們有一個以上的「 *開」和「關閉* 」狀態，實際上會回應其壓力的程度。</span><span class="sxs-lookup"><span data-stu-id="9c92f-192">These buttons are analog controls, meaning they have more than simple *on and off* states, they actually respond to the degree of pressure put on them.</span></span> <span data-ttu-id="9c92f-193">如此一來，我們就可以像 **天生杆** 一樣地建立類似的結構。</span><span class="sxs-lookup"><span data-stu-id="9c92f-193">This gives us a similar construct as the **thrust lever** .</span></span> <span data-ttu-id="9c92f-194">與原始遊戲和手手勢不同的是，當使用者停止對觸發程式造成壓力時，此控制項將會剪下出貨的天生。</span><span class="sxs-lookup"><span data-stu-id="9c92f-194">Unlike the original game and the hand gesture, this control will cut the ship's thrust once a user stops putting pressure on the trigger.</span></span> <span data-ttu-id="9c92f-195">它仍然提供使用者與原始 arcade 遊戲一樣的 finesse 程度。</span><span class="sxs-lookup"><span data-stu-id="9c92f-195">It still gives the user the same degree of finesse as the original arcade game did.</span></span>

<span data-ttu-id="9c92f-196">![左邊的操縱杆對應到偏擺和變換，右邊的操縱杆對應到音調和變換](images/thumbsticksidebyside.gif)</span><span class="sxs-lookup"><span data-stu-id="9c92f-196">![Left thumbstick is mapped to Yaw and Roll, Right thumbstick is mapped to Pitch and Roll](images/thumbsticksidebyside.gif)</span></span><br>
<span data-ttu-id="9c92f-197">*左邊的操縱杆對應至偏擺和變換;右邊的操縱杆對應到音調和變換*</span><span class="sxs-lookup"><span data-stu-id="9c92f-197">*Left thumbstick is mapped to yaw and roll; right thumbstick is mapped to pitch and roll*</span></span>

<span data-ttu-id="9c92f-198">雙重 thumbsticks 自然是控制出貨的旋轉。</span><span class="sxs-lookup"><span data-stu-id="9c92f-198">The dual thumbsticks naturally lend themselves to controlling ship rotation.</span></span> <span data-ttu-id="9c92f-199">可惜的是，出貨可以旋轉3個軸，而且兩個 thumbsticks 都支援兩個軸。</span><span class="sxs-lookup"><span data-stu-id="9c92f-199">Unfortunately, there are 3 axes on which the ship can rotate and two thumbsticks which both support two axes.</span></span> <span data-ttu-id="9c92f-200">這種不相符的情形是，其中一個操縱杆控制一個軸;或 thumbsticks 的座標軸重迭。</span><span class="sxs-lookup"><span data-stu-id="9c92f-200">This mismatch means either one thumbstick controls one axis; or there is overlap of axes for the thumbsticks.</span></span> <span data-ttu-id="9c92f-201">先前的解決方案會覺得「中斷」，因為 thumbsticks 本質上是混合其區域 X 和 Y 值。</span><span class="sxs-lookup"><span data-stu-id="9c92f-201">The former solution ended up feeling "broken" since thumbsticks inherently blend their local X and Y values.</span></span> <span data-ttu-id="9c92f-202">後者的解決方案需要進行一些測試，以找出哪些多餘的軸最自然的感覺。</span><span class="sxs-lookup"><span data-stu-id="9c92f-202">The latter solution required some testing to find which redundant axes feel the most natural.</span></span> <span data-ttu-id="9c92f-203">最後一個範例會使用 *偏擺* 和 *變換 (的 Y 和 x* 軸) 適用于左邊的操縱杆，以及針對右邊的 *操縱杆，將 (Z* *軸和 x* 軸) 。</span><span class="sxs-lookup"><span data-stu-id="9c92f-203">The final sample uses *yaw* and *roll* (Y and X axes) for the left thumbstick, and *pitch* and *roll* (Z and X axes) for the right thumbstick.</span></span> <span data-ttu-id="9c92f-204">這認為最自然的情況 *是，* 與 *偏擺* 和 *推銷* 之間的表現完全獨立。</span><span class="sxs-lookup"><span data-stu-id="9c92f-204">This felt the most natural as *roll* seems to independently pair well with *yaw* and *pitch* .</span></span> <span data-ttu-id="9c92f-205">請注意，同時使用這兩個 thumbsticks 來進行 *變換也會* 發生兩倍的旋轉值;使用 lander do 迴圈相當有趣。</span><span class="sxs-lookup"><span data-stu-id="9c92f-205">As a side note, using both thumbsticks for *roll* also happens to double the rotation value; it's pretty fun to have the lander do loops.</span></span>

<span data-ttu-id="9c92f-206">這個範例應用程式會示範空間辨識和 tactile 深度如何大幅改變體驗，因為這是 Windows Mixed Reality 的可延伸輸入形式。</span><span class="sxs-lookup"><span data-stu-id="9c92f-206">This sample app demonstrates how spatial recognition and tactile immersion can significantly change an experience thanks to Windows Mixed Reality's extensible input modalities.</span></span> <span data-ttu-id="9c92f-207">雖然陰曆 Lander 的年齡可能接近40年，但以幾乎的八邊形來公開的概念將會無限期地存留。</span><span class="sxs-lookup"><span data-stu-id="9c92f-207">While Lunar Lander may be nearing 40 years in age, the concepts exposed with that little octagon-with-legs will live on forever.</span></span> <span data-ttu-id="9c92f-208">想像未來，為什麼看不到過去？</span><span class="sxs-lookup"><span data-stu-id="9c92f-208">When imagining the future, why not look at the past?</span></span>
